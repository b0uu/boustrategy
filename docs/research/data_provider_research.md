# Data provider research

Checked 2026-07-10. Pricing and access terms change quickly; re-verify this report if the decision is made after 2026-09-10.

## 1. TL;DR + recommendation

Use a two-stage X path: run a 2-4 week manual-curation MVP first, then automate with the official X API under a hard monthly spend cap. The official API now charges per resource rather than requiring the old fixed Basic subscription: a public Post read is $0.005 and a User read is $0.010, with 24-hour deduplication ([X pricing](https://docs.x.com/x-api/getting-started/pricing), accessed 2026-07-10). That makes a narrow 10-50-handle feed economically plausible and preserves the raw post, author, timestamp, and Post ID needed by the scrutiny ledger. Keep xAI X Search as an optional semantic-triage experiment, not the audit source: it can restrict a search to 20 named handles and return citations, but its primary output is a model response rather than a deterministic, exhaustive raw-post stream ([xAI X Search](https://docs.x.ai/developers/tools/x-search), accessed 2026-07-10).

Do **not** buy Unusual Whales for v0.1. Its official MCP is real and broad, but options flow is a positioning/crowding input adjacent to BouStrategy's thesis, not the curated-human-narrative input at the center of it. Revisit it only after the core thesis workflow is operating and an eval can show that a daily crowding feature improves decisions. The recommended stack has no external data charge during the manual MVP; after automation, cap X at **$20/month** and keep the remaining sources on their documented free tiers. The $20 ceiling is an operator-set cap, while the underlying X price is the official $0.005 per Post resource ([X pricing](https://docs.x.com/x-api/getting-started/pricing), accessed 2026-07-10).

Ranked path:

1. **Manual curated-X MVP** — 15 minutes/day for 2-4 weeks; no external provider charge, highest learning value, and it creates labeled examples for later evals.
2. **Official X API pay-per-use** — production ingestion once the handle list and daily useful-post volume are measured; raw, attributable, and the cleanest terms posture.
3. **xAI X Search pilot** — inexpensive discovery/triage around the same curated handles, but only alongside raw source links and never as the sole scrutiny-ledger evidence.
4. **Third-party X data providers** — prototype-only at most; do not put their hydrated content on the public dashboard without legal/terms review.
5. **Unusual Whales MCP** — later, conditional on a crowding-detection eval; not a substitute for X.

#### Official X API details

The official API is no longer the historical flat Basic-plan decision. It is credit-based pay-per-use, bills reads per returned resource, and deduplicates the same resource within a UTC day ([pricing](https://docs.x.com/x-api/getting-started/pricing), accessed 2026-07-10). The relevant paths are:

- Resolve up to 100 curated usernames per lookup request, then retain stable User IDs ([user lookup](https://docs.x.com/x-api/users/lookup/introduction), accessed 2026-07-10).
- Read each user's timeline; the documented ceiling is the 3,200 most recent Posts and the endpoint rate limit is 10,000 app requests per 15 minutes, far above this project's needs ([timeline guide](https://docs.x.com/x-api/posts/timelines/integrate), [rate limits](https://docs.x.com/x-api/fundamentals/rate-limits), accessed 2026-07-10).
- Use recent search for the last seven days or full-archive search when needed; `from:handle` is supported, recent search returns up to 100 Posts/request, and full archive up to 500 ([search overview](https://docs.x.com/x-api/posts/search/introduction), accessed 2026-07-10).

For the public dashboard, store Post IDs/URLs and BouStrategy's own claim summaries. Rehydrate or embed current X content when displaying it; do not publish a downloadable archive of hydrated Posts. X requires removal of deleted content and restricts redistribution ([Developer Policy](https://docs.x.com/developer-terms/policy), accessed 2026-07-10).

#### xAI / Grok details

X Search can constrain the model to `allowed_x_handles` (maximum 20), constrain dates, search threads, and emit citations ([X Search docs](https://docs.x.ai/developers/tools/x-search), accessed 2026-07-10). It can therefore cover 50 handles in three batches and is attractive for "what changed today?" triage. The fit gap is structural: the response is generated prose, and the model decides which searches and citations to surface. It should never silently turn "not mentioned" into "not posted," and every accepted claim should retain a cited Post URL plus independently captured author/time fields.

#### Third-party and manual-MVP posture

SocialData and TwitterAPI.io are inexpensive enough that cost is not the decision. Reliability and rights are. X's developer guidance says to use only the official API and treats scraping/non-API automation as prohibited ([X guidelines](https://docs.x.com/developer-guidelines), accessed 2026-07-10). That makes an unofficial provider a poor foundation for a transparent public investment dashboard even when the provider says it handles only public data.

The manual MVP should run for 2-4 weeks with a fixed paste schema and a daily cap (for example, the 5-15 most decision-relevant Posts). It will measure useful-Post volume, expose whether 10, 20, or 50 handles are actually manageable, and create labeled positive/negative examples for ingestion recall, scrutiny-ledger scoring, and later reasoning evals.

The MCP question has a positive factual answer: an **official** server exists, inspired by an earlier community server. It supports a hosted endpoint and a local `@unusualwhales/mcp` package, requires an Unusual Whales API key, advertises 100+ endpoints, and sets a client-side default of 120 requests/minute ([official MCP README](https://github.com/unusual-whales/unusual-whales-official-mcp), accessed 2026-07-10). The public API catalog confirms REST, WebSocket, Kafka, and MCP access and enumerates options, dark-pool, congress, news, fundamentals, and other tools ([API docs](https://api.unusualwhales.com/docs), accessed 2026-07-10).

What it means for BouStrategy:

- **Options flow / GEX / dark pools:** strongest doctrinal hook is SB-005 crowding and late-consensus detection. Treat these as positioning evidence, not directional truth; a large print can be opening, closing, hedging, or one leg of a multi-leg trade.
- **Congress trading:** useful slow event/disclosure source, closest to `NEWS` in the current vocabulary, not a live edge by itself because disclosures lag.
- **News and alerts:** map to `NEWS`; economic endpoints map to `MACRO`; company filings/fundamentals should preserve the ultimate SEC or company source where available rather than laundering provenance through the aggregator.
- **Daily long-only fit:** end-of-day aggregate crowding features could improve trim/add scrutiny. Intraday flow alerts are a poor match because no current state machine acts intraday.

If adopted, add a new `MARKET_POSITIONING` source type (with subtypes such as `OPTIONS_FLOW`, `DARK_POOL`, `GEX`, and `SHORT_DATA`) rather than forcing flow into `PRICE_DATA`. Keep congress disclosures under `NEWS` for v0.1, or introduce `GOVERNMENT_DISCLOSURE` only if several government disclosure sources appear. The decision should be gated by an eval: does a daily positioning feature reduce adds into crowded/late setups without suppressing good thesis-led entries?

Pricing gap: the public pricing pages were searched and fetched at both `/pricing?product=api` and `/public-api#pricing`; they returned product directions but no numeric token tiers without entering the pricing/account flow. No account, trial, payment, or contact step was attempted. The exact current API-token price is therefore explicitly **UNANSWERABLE** in this environment.

The no-cost floor is already enough for v0.1: SEC for filings, FRED for macro, yfinance cached locally for daily OHLCV, and Alpha Vantage or Massive as a structured fallback. Paid market data should wait for observed outages, adjustment errors, or licensing needs. Finnhub is worth a narrow comparison if the project later wants one broad vendor, but there is no reason to pay merely to duplicate daily bars today.

## 3. Open decisions for the maintainer

1. **Manual-first duration:** approve a fixed 2- or 4-week manual X run. Two weeks gets to automation sooner; four weeks captures more earnings/news regimes and creates a better labeled eval set.
2. **Official X monthly cap:** choose a hard cap, recommended $20/month, and an alert threshold below it. At the official $0.005/Post price, the cap covers about 4,000 unique Post reads/month; a lower cap forces stronger handle/query discipline ([X pricing](https://docs.x.com/x-api/getting-started/pricing), accessed 2026-07-10).
3. **Raw ingestion versus xAI triage:** decide whether v0.1 ingests every new Post from curated timelines or asks xAI to shortlist cited Posts first. Raw ingestion maximizes auditability/recall; xAI reduces reading volume but introduces model-controlled omissions.
4. **Public-dashboard X content:** decide whether to display only BouStrategy claim summaries plus source links/IDs, or embed/rehydrate Posts. Summaries plus links minimize deletion/redistribution complexity; embeds preserve context but require a content-removal process.
5. **Third-party prohibition:** decide whether unofficial X providers are categorically disallowed or permitted only in throwaway local experiments. A categorical ban is the cleanest match to the public-dashboard ambition and X's anti-scraping guidance.
6. **Unusual Whales timing:** choose `later` (recommended) or `never for this strategy`; do not choose `now` without exact API pricing and an eval design. `Later` preserves the crowding-control option, while `never` protects focus and budget.
7. **New source type if flow is adopted:** approve `MARKET_POSITIONING` or require all signals to fit the existing vocabulary. A dedicated type makes provenance and policy clearer; reusing `PRICE_DATA` avoids migration but conflates executed prices with inferred positioning.
8. **Price fallback:** choose Alpha Vantage or Massive as the first supported fallback after yfinance. Alpha Vantage's 25 requests/day is adequate for a concentrated daily portfolio; Massive offers a broader structured surface at 5 requests/minute.
